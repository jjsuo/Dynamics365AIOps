# K04 — 领域知识生成（AI 辅助，人工审核）

## 目标
为每个「领域 + solution」组合生成一份领域知识 Skill 文档。
有 N 个领域 × M 个 solution 时，跑 N×M 次。

## 输入
- `.dkf/domains/K03_confirmed.json`（确认后的领域）
- 对应实体的字段：用扫描器的 `scan_entity_attributes(logicalname)` 按需取
- `.dkf/meta/` 下相关入口/流程/安全

## 每份 Skill 文档结构（写入 .dkf/skills/<domain>.md）
1. **域概述**：这个域解决什么业务问题，边界在哪
2. **核心实体**：每个实体的作用 + 关键自定义字段（指向 meta，不复述全部字段）
3. **关键业务流程**：BPF / flow / 经典工作流，按业务链路串起来
4. **安全模型**：哪些角色能读写这个域
5. **集成点**：插件/Custom API/连接器（指向 K01_A、K01_B）
6. **坑点与约定**：实施时容易踩的、客户特有的业务规则

> 核心原则：写"从哪看、看什么"，不复制元数据。逻辑判断回到 Dataverse。
